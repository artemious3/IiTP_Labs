## Общее описание системы

Онлайн-маркетплейс -- система, основной задачей которой является транспортировка на 
большие расстояния товара между производителем и клиентом.

Жизненный цикл заказа имеет следующий вид:

```
Заказ не подтвержден : в процессе выбора товаров, пункта выдачи
                🠋    
Заказ подтверждён : клиент указал окончатьную информацию
                🠋    
Заказ маршрутизирован : логист построил оптимальный маршрут между складом производителя
                        и пунктом выдачи
                🠋    
Заказ на пути на склад 
                🠋    
Заказ на складе и ждёт отправки 
                🠋    
               ...                  (заказ перемещается между складами)
                🠋    
Заказ в пункте выдачи и ждёт получения
                🠋    
Заказ завершён : оплачен и получен клиентом
```

## Описание сущностей и логики их функионирования


ЗАМЕЧАНИЕ: если не сказано обратное, поле НЕ может быть NULL

### Пользователь (User)

Лицо, взаимодействующее с проектируемой информационной системой.
Содержит классический набор персональных данных,
в частности, данные аунтефикации и контактные данные (см. диаграмму)

- id: идентификатор

- login: логин

- password: пароль

- first_name: имя

- last_name: фамилия

- phone: телефон

- email: электронная почта

- birth_date: дата рождения

- role: роль

### Клиент (Client)

Функциональные требования к ролям описаны выше

- id: идентификатор

- user_id: идентификатор пользователя

### Логи (Logs)

Журнал действий пользователя (см. диаграмму)

- id: идентификатор записи

- user_id: идентификатор пользователя

- time: время

- text: текст


### Способ оплаты (PaymentMethod)

Способ оплаты заказа (банковская карта), с помощью которой могут быть оплачены заказы (см. диаграмму)

 - payment_method_id: идентификатор способа оплаты

 - client_id: идентификатор клиента

 - token: токен

 - card_type: тип карты

 - last_four_digits: последние четыре цифры

 - expiration_date: дата истечения срока действия

 - is_default: по умолчанию

 - created_at: дата создания

 - updated_at: дата обновления

### Заказ 

Сущность, объединяющая множество товаров, заказанных клиентом для последующего получения. 

До завершения заказа, находится в одном из 2-х состояний : хранится на складе, либо 
перемещается между ними. 

Заказ является маршрутизированным (is_routed) только если логист создал маршрут для этого заказа.

Оплата заказов может производиться как сразу, так и при получении. 

Заказ является выполненным (is_completed) только если он оплачен И получен клиентом.

- id: идентификатор заказа

- client_id: идентификатор клиента

- is_routed: маршрутизирован

- is_paid: оплачен

- is_completed: выполнен

- current_warehouse : склад, в котором находится заказ  (NULLABLE)

- target_warehouse : склад, в который направляется заказ (NULLABLE)

ЗАМЕЧАНИЕ: *Одно из двух полей current_warehouse и target_warehouse не NULL*

- dropsite: пункт выдачи

### Товар (Product)

Товар, предлагаемый клиенту для добавления в заказ и последующей оплаты и получения вместе с заказом.
( см. диаграмму)

- id: идентификатор

- title: название

- description: описание

- mass: масса 

- price: цена

- producer_id: идентификатор производителя

### Производитель (Producer)

Юридическое лицо, отвечающее за предоставление товара маркетплейсу. Обязательно имеет свой склад, который 
является начальной точкой маршрутов заказов.
(см. диаграмму)

- id: идентификатор

- name: название

- warehouse_id: идентификатор склада

### Связь товара и заказа (OrderProductRelation)

Промежуточная сущность, реализующая связь ManyToMany с информацией о количестве товара в заказе
(см. диаграмму)

- order_id: идентификатор заказа

- product_id: идентификатор продукта

- amount: количество


### Склад (Warehouse)

Сущность, являющаяся начальной, промежуточной и конечной точка маршрута каждого заказа. 
На кажом этапе заказ ЛИБО находится на складе, ЛИБО направляется на склад.

Ниже перечислены 3 типа складов:

 - Склад производителя -- начальная точка заказа.

 - Склад маркетплейса (условно сортировочный пункт) -- промежуточная точка заказа. 

 - Пункт выдачи -- конечная точка заказа. 

Только работники пункта выдачи имеют права подтвердить получение заказа клиентом.

(см. диаграмму)

Поля: 

- id: идентификатор склада

- location_id: идентификатор местоположения

- is_dropsite: является пунктом выдачи

### Место (Location)

Сущность, хранящая географические координаты и адрес некоторого места.
(см. диаграмму)

 - id: идентификатор местоположения

 - coordinates: координаты

 - address: адрес


### Пункты маршрута заказа (OrderRoutePoints)

Сущность, реализующая связь ManyToMany между заказом и складом. Определяет маршрут,
построенный логистом от склада производителя, до выбранного клиентом пункта выдачи. 
Последовательность определяется полем Index.

(см. диаграмму)

- order_id: идентификатор заказа

- warehouse_id: идентификатор склада

- index: индекс (порядковый номер)

- is_order_here: здесь находится заказ
